# RCT v1 Implementation Plan

> For agentic implementation: use `test-driven-development` and `subagent-driven-development` skills.

## Goal

Implement the full RCT v1 config-driven hook framework as specified in `2026-03-28-rct-v1-design.md`.

## Architecture

Config-driven hook runner with typed API. Single entry point script loads config, evaluates rules/injections/files for the current event, and outputs JSON to stdout. CLI tool generates config and wires settings.json.

## Tech Stack

- Bun, TypeScript (strict), @anthropic-ai/claude-agent-sdk

  ```typescript
  import { existsSync, readFileSync } from "fs"
  import path from "path"
  import type { RCTConfig } from "./types"

  export const CLAUDE_PROJECT_DIR = process.env.CLAUDE_PROJECT_DIR ?? process.cwd()

  const CONFIG_FILES = ["rct.config.ts", "rct.config.js", "rct.config.json"] as const

  export async function loadConfig(root = CLAUDE_PROJECT_DIR): Promise<RCTConfig> {
    for (const filename of CONFIG_FILES) {
      const configPath = path.resolve(root, filename)
      if (!existsSync(configPath)) continue
      if (filename.endsWith(".json")) {
        return JSON.parse(readFileSync(configPath, "utf-8"))
      }
      const mod = await import(configPath)
      return mod.default ?? mod
    }
    return {}
  }
  ```
